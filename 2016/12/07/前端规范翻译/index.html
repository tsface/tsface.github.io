前端规范

Posted on 2016-12-07 Edited on 2019-12-06

Front end Coding specification

HTML

  • Grammer
    • Disallow add ==/== at the end of self-closing tag eg. <br>
    • Disallow omitted closing tag
    • Using "" to define element attribute

  • Character Encoding
    <meta charset="UTF-8">

  • IE compatible mode

    • Specify IE version
    • Using Edge mode if no special requirment, it will use the newest IE mode
      <meta http-equiv="X-UA-Compatible" content="IE=Edge">

  • Link css between head tag to rendering dom first

  • Important JavaScript files at the end of body tag for loading js will block rendering dom

    CSS

  • Grammer

    • Add a space begin of left =={==
    • keep ==}== as a single line
    • One statement one line, end with semicolon
    • Using lower case and shortening to define color, eg. #fff
    • Not assign unit for 0, eg. write ==margin:0;== instead of ==margin:0px;==
    • Separate concatenation selector

  • Meaningful comment

  • Name

    • Using low case charactor and dashe(like name space) to name, ensure short length and meaningful
    • Disallow using Hump nomenclature
    • Using parent class or base class as the new class prefix instead of descendant selector

  • Selector

    • Using class selector
    • The number of combination selector is better less than three eg. .nav .container .header
    • Not use !important, according to CSS properity to deal.
      !important -> style -> id selector -> class selector, attribute selector, pseudo-classes selector, pseudo-attribute selector -> element selector -> *

  • Declaration order

    • Positioning
    • Box model
    • Typographic
    • Visual

JavaScript

  • Statement valable with var, for the scope of str = 123 is global, but var str = 123 is local
  • Using call() or apply() to change this instead of define global valable eg. var vm = this
  • Using === instead of == to avoid implicit conversion
  • Using isNaN() when checking for NaN
  • Write semicolon at end of expression
  • Disallow constant expressions in conditions
  • Disallow calling some Object.prototype methods directly on objects
